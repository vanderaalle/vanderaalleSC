General usage


Let's suppose we have to describe the default SC audio example contained here:


"sounds/a11wlk01.wav" 


1. Loading a file


o = SO.new("sounds/a11wlk01.wav") ;


This creates a new SO object and loads a sound file.

Some information are accessible by evaluating:


o.duration ;

o.soundPath ;


If you want to listen to it, you can simply do:


o.play ;


This creates a new SO object and loads a sound file. Then, you can open the same file e.g. with Praat (Read ->Read from file ->Edit), so that you can explore sonogram interactively. 

At first hear, three sound objects can be recognized:


1. a background noise

2. a couple of beep (at the beginning and at the end)

3. a child voice


Apart from a little decrease in dynamics corresponding to the beeps, background noise is constant.

So we consider it as a unique sound object, which attack is 0 and duration coincides with sound file duration

attack: 0

duration: o.duration (Praat would say: 4.283288)

We can add a first sound object, named "background":


o.add("background", 0, o.duration) ;

Then we can add the beeps. Attacks and duration can be retrieved with Praat. The first one:


o.add("beep1", 0.123, 0.35) ; 


You can hear it by evaluating:


o.play("beep1") ; 


You may not be satisfied with the time boundaries. The easiest way is to interactively test by simply re-adding the sound object and playing it. Re-adding overwrites the previous sound object (there cannot be two sound objects with the same name).


o.add("beep1", 0.123, 0.32) ; 

o.play("beep1") ; 

o.add("beep1", 0.123, 0.3) ; // that 's it 

The same goes for the second beep:


o.add("beep2", 3.8, 0.37) ; 

o.play("beep2") ; 


If you want to see what is the current state of your objectDictionary:

o.print // prints all

o.print("beep1") // the one you want 


The difficult case is the voice. There are two possible criteria:

- to consider the whole utterance as a unique sound object. This would respect the figurative labelling (background, beeps, voice). More, the voice presents homogeneous properties, at least if compared to the other two sound object types.

- to decompose the voice. The rationale is that the fricative block is very different typologically from the other syllables. More, the syllables clearly respects the articulation/sustain criterium requested by Schaeffer (to be precise: the criterium is modelled after voice).


Following the secon criterium, we may have:


o.add("[kA'-l:m]", 0.35, 0.71) ; 

o.add("[bi-e']", 1.05, 0.65) ; 

o.add("[tSys]", 1.91, 0.6) ; 

o.add("[der]", 2.8, 0.23) ; 

o.add("[ovA]", 3.19, 0.6) ; 

Names are given following a very approximative phonetic transcription (UTF support in SuperCollider would be needed, but labels are just intended as aides-mémoires).

Still, we have not provided typo-morphological information. This is evident by inspecting the dictionary containing the objects:

o.print


Even if it is possible to immediately provide  values for all the parameters, it is easy to assign them by exploring the typological space. This can be done by evaluating:


p = SOSpace.new(o) ; 


The sound objects are represented by squares. The three views represent the upper, front and lateral side of the typological space, like in a flattened paper box. They  are indeed correlated: moving a sound object in a view determines a related moving in one of the other two views.

The typological space is defined by profile, caliber, variation.  As we have still not provided the relevant typological information, a random position is assigned to sound objects and they are randomly scattered all over the space. By clicking with mouse on one of them, it gets selected, and its definition appears in the info panel. Changing the position by dragging it updates the info panel. The contrary applies too. If you change the value on the info panel and press Return, changes are stored and -if needed- the position in the space is updated. 

Through the info panels the values of all parameters can be assigned. More, the entry widget at the right of "Name" allows to rename the sound object.

As we have not provided a group index, all objects are by default grouped together in the group 0. 

Groups are integers which allows for useful grouping, i.e. like voices or layers. 

We can identify three groups by assigning a different group index: the background, the beeps, the vocal utterances. Group indices must not be consecutive, they only have to be different integers (e.g. 3, 45, 123 is fine, even if a bit eccentric).


The same can  be obtained by evaluating:


(

o.changeGroup("background", 0) ;

o.changeGroup("[ovA]", 1) ;

o.changeGroup("[tSys]", 1) ;

o.changeGroup("[der]", 1) ;

o.changeGroup("[bi-e']", 1) ;

o.changeGroup("[kA'-l:m]", 1) ;

o.changeGroup("beep1", 2) ;

o.changeGroup("beep2", 2) 

)


Background is noisy, so its caliber tends to be large, but not maximum (i.e. white noise), so 1.7. More, its site (lowest register) tends to be placed in the low register (0). Dynamically, it is ppiano (2). Variation is null (0).  Sustain is sustained (.e. continuous), negative. Profile is strictly amorphous (but not artificial, hence -2.4).

Note that the previous text can be inserted in the text field.

If using textual interrface, in order not to deal with temporal information we can use changeFeatures:


o.changeFeatures("background", -2.4, 1.7, 0.0, 0, 2, 0, "") ; 

The beeps are substantially two repetitions of the same object. They are surely pitched, so their mass is notal (0.0 < caliber < 1.0): we can assign 0.2. They are continuous, so sustain is negative, and they are amorphous, but due to short duration they do not reach the maximum (i.e. in some way they are still "notes"): hence -2. Variation is null (0). Site is very acute (6). Dynamics is a bit greater tha background, so 2.

We have:


o.changeFeatures("beep1", -2, 0.2, 0.0, 6, 3, 2, "") ; 

o.changeFeatures("beep2", -2, 0.2, 0.0, 6, 3, 2, "") ; 


Vocal sounds are in acute register (site: 5), with limited caliber (0.3), apart  [tSys], distinclty more noisy, falling in the second half of caliber range (1.1). Their dynamics, at least if compared ti the other groups is forte and substantially homogeneous (5): they are distinctively in the foreground. They are continuous, and typically notal (profile = -1.0). Variation differs in each sound object, the first two being tow distinct glissandos, [der] and [ova] beign almost static, [tSys] being a portando over approx a fourth, but with a clear sustained midpoint. Hence:


o.changeFeatures("[kA'-l:m]", -1, 0.3, 0.7, 5, 5, 1, "") ; 

o.changeFeatures("[bi-e']", -1, 0.3, 0.7, 5, 5, 1, "") ; 

o.changeFeatures("[tSys]", -1, 1.1, 0.5, 5, 5, 1, "") ; 

o.changeFeatures("[der]", -1, 0.3, 0.2, 5, 5, 1, "") ; 

o.changeFeatures("[ovA]", -1, 0.3, 0.3, 5, 5, 1, "") ; 


It is possible to save the data by evaluating:


o.save("/typoTest.txt") // the name and the path you prefer


Symmetrically, a file can be open by evaluating: 

o = SO.new("sounds/a11wlk01.wav") ; // if there's no SO

o.open("/typoTest.txt") // the name and the path you prefer

o.print ;


The SOSpace shows a concentration on the right, continuous and the frontal, constant sides of the space.

Finally, we can draw a score exploiting the temporal information.

Simply:

c = SOScore(o)
